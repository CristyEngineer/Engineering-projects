# 📘 Lean Six Sigma Project — DMAIC & DMADV
Process Design and Process Improvement in an IT Environment

This Lean Six Sigma project was carried out within an IT service organization. The project combines both DMADV (designing a new process) and DMAIC (improving an existing process), applied to a complex onboarding workflow for cloud‑based workplace environments. 

# 🎯 Project Summary
The goal of the project was to:
- Improve an existing operational process using DMAIC
- Design a new, standardized and scalable process using DMADV
- Reduce variation, rework and lead time
- Increase quality, predictability and customer satisfaction
The project was executed in collaboration with multiple stakeholders across engineering, operations and service teams.

# 🧩 Methods Used
DMADV — Process Design: 
- Define: problem definition, stakeholder alignment, project charter
- Measure: VOC, CTQ definition, SIPOC, baseline data
- Analyze: root cause analysis, statistical testing, House of Quality
- Design: new standardized workflow, templates, automation concepts
- Verify: pilot, validation, KPI monitoring

DMAIC — Process Improvement: 
- Define: context, scope, risks, business case
- Measure: systematic data collection, MSA, baseline performance
- Analyze: ANOVA, Pareto, 5 Why, Ishikawa
- Improve: solution generation, selection matrix, implementation
- Control: control plan, dashboards, process governance

# 📊 Learning Outcomes Demonstrated
- Designing and improving processes using Lean Six Sigma
- Creating a complete project charter with context, risks and business case
- Translating customer needs into measurable CTQ requirements
- Performing statistical analyses to identify root causes
- Selecting and validating solutions based on data
- Documenting and standardizing workflows
- Reflecting on professional development as an engineer
